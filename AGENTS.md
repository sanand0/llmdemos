# Repo Automation

Always prefix shell commands with `rtk`.

Use [`scripts/sync_github_demos.py`](/home/vscode/code/llmdemos/scripts/sync_github_demos.py) for GitHub-backed updates.

## Requirements

- `GITHUB_TOKEN` must be available in the shell or `.env`
- Run from the repo root: `/home/vscode/code/llmdemos` or `/home/sanand/code/llmdemos`

## Common Commands

Update `config.json` with `branded: true` when repo contents mention any of:
`Straive`, `Gramener`, `Learning Mate`, `Double Line`, `SG Analytics`

```bash
rtk uv run scripts/sync_github_demos.py brand-config --write
```

List repo details for specific repos:

```bash
rtk uv run scripts/sync_github_demos.py repo-info sanand0/tools ritesh17rb/chart-map
```

Discover GitHub Pages repos for one or more users using the latest `created` date in `config.json` as the default cutoff:

```bash
rtk uv run scripts/sync_github_demos.py discover-pages nitin399-maker pavankumart18 ritesh17rb mynkpdr
```

Use a manual cutoff and include repos already present in `config.json`:

```bash
rtk uv run scripts/sync_github_demos.py discover-pages sanand0 --cutoff 2026-03-01 --include-existing
```

## Adding New Demos

Most updates start by discovering public GitHub repos with GitHub Pages and then manually adding only the good demos to `config.json`.

### 1. Find the cutoff date

Use the latest non-empty `created` value already in `config.json`.

```bash
rtk jaq -r '.demos | map(.created) | max' config.json
```

### 2. Discover candidate GitHub Pages repos

Run discovery for the requested owners. If `--cutoff` is omitted, the script uses the latest `created` value from `config.json`.

```bash
rtk uv run scripts/sync_github_demos.py discover-pages USER1 USER2 USER3
```

Use `--include-existing` only when auditing or refreshing existing entries.

```bash
rtk uv run scripts/sync_github_demos.py discover-pages USER1 --cutoff 2026-03-01 --include-existing
```

### 3. Review candidate quality

Add only good demos. A good demo should have a working public URL and enough substance to be useful as a demo, report, benchmark, dashboard, tool, or interactive story.

Skip candidates that are:

- 404 or blocked at the published URL
- placeholder pages with no loaded data or no meaningful content
- personal tribute/profile pages
- README-only redirects with no real demo surface
- broken builds, missing assets, or pages that require private credentials just to see the core value

Useful review commands:

```bash
rtk w3m -dump https://OWNER.github.io/REPO/
rtk curl -Ls https://raw.githubusercontent.com/OWNER/REPO/main/README.md
rtk uv run scripts/sync_github_demos.py repo-info OWNER/REPO
```

### 4. Add entries to `config.json`

Add new demos under `.demos`, normally near the top in descending `created` order.

Required fields:

- `icon`: Bootstrap icon class, usually `bi-*`
- `title`: short display title
- `url`: public demo URL
- `description`: one concise sentence explaining what the demo does
- `reviewed`: set to `false` for newly discovered demos needing human review
- `created`: GitHub Pages first build date from discovery, formatted `YYYY-MM-DD`
- `license`: SPDX license from GitHub, or `""` if absent
- `repo`: canonical GitHub repo URL ending with `/`

Optional fields:

- `branded`: `true` when repo contents mention configured brand terms
- `brands`: evidence generated by `brand-config`

Example:

```json
{
  "icon": "bi-bar-chart-line",
  "title": "Demo Title",
  "url": "https://owner.github.io/repo/",
  "description": "Interactive demo that explains the core user-facing value in one sentence.",
  "reviewed": false,
  "created": "2026-04-13",
  "license": "MIT",
  "repo": "https://github.com/owner/repo/"
}
```

Do not add both `reviewed` and an inverse `unreviewed` field. Use only `reviewed`.

### 5. Refresh generated files

After editing `config.json`, run the same steps as `just build`:

```bash
rtk uv run scripts/sync_github_demos.py brand-config --write
rtk bash -lc 'npx -y mustache config.json template.html > index.html'
rtk uv run scripts/generate_demos_csv.py
```

`scripts/generate_demos_csv.py` writes:

- `llmdemos.csv`: all demos
- `open-demos.csv`: reviewed MIT-licensed repos only

### 6. Validate before finishing

```bash
rtk jaq empty config.json
rtk git diff --check -- config.json
rtk git diff --stat
```

Confirm unrelated local changes are not reverted or mixed into the update.

## Notes

- The helper dedupes repos from `config.json` before scanning.
- Branding checks are cached in `.cache/github-repo-helper.json` to make reruns cheap.
- `brand-config` adds `branded: true` only for matching repos and removes stale `branded` flags from non-matches.
- After config updates, rebuild generated outputs with:

```bash
rtk bash -lc 'npx -y mustache config.json template.html > index.html'
rtk uv run scripts/generate_demos_csv.py
```
